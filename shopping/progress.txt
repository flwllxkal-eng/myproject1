# 쇼핑몰 프로젝트 진행 상황 및 계획

이 문서는 쇼핑몰 기능 개발의 현재 상태와 향후 계획을 기록합니다.

## 완료된 작업 (2024년 10월 27일)

1.  **기본 파일 구조 생성:**
    *   `shopping/` 디렉터리 및 기본 파일들(`index.html`, `style.css`, `main.js`)을 생성했습니다.

2.  **메인 페이지에 쇼핑몰 홍보 섹션 추가:**
    *   루트 `index.html`에 쇼핑몰을 소개하는 눈에 띄는 'hero' 섹션을 추가했습니다.
    *   이 섹션은 그라데이션 배경과 매력적인 버튼을 포함하여 사용자의 시선을 끌도록 디자인되었습니다.
    *   `style.css`에 해당 섹션과 버튼에 대한 스타일을 추가했습니다.

3.  **쇼핑몰 기본 UI 및 기능 구현:**
    *   기본적인 UI(헤더, 푸터, 제품 카드) 및 장바구니 추가 기능을 구현했습니다.

4.  **상품 데이터 분리 및 동적 로딩:**
    *   `products.json` 파일을 생성하여 상품 데이터를 분리했습니다.
    *   `shopping/main.js`를 수정하여 `fetch` API를 통해 비동기적으로 `products.json` 파일의 데이터를 불러오도록 변경했습니다. 이로써 데이터 관리가 용이해졌습니다.

## 수정된 파일

*   `./index.html`: 쇼핑몰을 위한 'hero' 섹션 추가 및 기존 카드 링크 제거.
*   `./style.css`: 'hero' 섹션 및 버튼에 대한 스타일 추가.
*   `shopping/index.html`: 생성 및 기본 구조 작성.
*   `shopping/style.css`: 생성 및 기본 스타일 작성.
*   `shopping/main.js`: 하드코딩된 데이터를 제거하고, `fetch`를 이용해 `products.json`에서 데이터를 불러오도록 수정.
*   `shopping/products.json`: 생성 및 상품 데이터 추가.
*   `shopping/progress.txt`: 생성 및 진행 상황 기록.

## 향후 계획 (다음에 할 일)

1.  **메인페이지에서 쇼핑몰 페이지로 가는 버튼 완성**

2.  **장바구니 페이지 구현:**
    *   `cart.html` 파일을 새로 생성합니다.
    *   장바구니에 담은 상품 목록을 표시하고, 수량을 조절하거나 삭제할 수 있는 기능을 추가합니다.

3.  **장바구니 상태 관리 개선:**
    *   현재는 페이지를 새로고침하면 장바구니가 초기화됩니다.
    *   `localStorage`를 사용하여 장바구니 상태가 브라우저에 유지되도록 개선합니다.

4.  **상품 상세 페이지 개발:**
    *   제품 카드를 클릭했을 때 해당 상품의 상세 정보를 볼 수 있는 `product-detail.html` 페이지를 구현합니다.

5.  **데이터 관리 심화:**
    *   JSON 파일을 넘어, 실제 API처럼 동작하는 Mock API 서버를 구축하여 데이터를 제공하는 방식으로 변경하는 것을 고려합니다.

6.  **UI/UX 개선:**
    *   전체적인 디자인을 다듬고, 애니메이션 효과나 더 나은 사용자 피드백을 추가합니다.
    *   제품 필터링 및 정렬 기능을 추가합니다.

7.  **결제 프로세스:**
    *   간단한 형태의 결제 폼 및 프로세스를 구현합니다. (실제 결제는 연동하지 않음)
